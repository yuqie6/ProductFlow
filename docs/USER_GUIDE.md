# ProductFlow 新手教程与参考说明

这份文档分两部分：

1. **新手教程**：少术语，照着点、照着填，先生成一张能用的商品图。
2. **参考说明**：跑通过一次后，再看工作台卡片、提示词配置、模型设置和常见问题。

产品内的新手入口是顶部导航右侧的 **开始引导/继续引导** 按钮；不再有单独的新手教程页面。

现在产品里已经有真正的 **guided onboarding（产品内引导）**：

- 顶部导航右侧点 **开始引导**；如果已经开始过，会显示 **继续引导**，系统会记住当前进度。
- 大块引导/进度面板只出现在 **商品/工作台首页（商品列表）**，不会占用新建商品页、工作台画布或连续生图页。
- 每一步都有当前进度、要做的动作、应该看到的结果和下一步按钮。
- 你可以随时 **标记完成**、**跳过** 或 **重置**；重置后会从“新建商品”重新开始。
- 这份 repo 文档只保留文字参考；实际操作时可以从首页的引导面板或顶部导航按钮继续。

## 新手教程：从一张商品图开始

目标：上传一张商品图，补一点资料，生成文案，再生成一张满意的图片。

### 1. 新建商品

1. 点顶部导航右侧 **开始引导**；如果已经开始过，会显示 **继续引导**。
2. 引导会先带你到 **商品/工作台**。
3. 点 **新建商品**。
4. 上传一张清楚的商品主图。
5. 填商品名，例如：`奶油白通勤托特包`。
6. 点 **创建并继续**。

你应该看到：页面进入这个商品的工作台，中间有几张可点击的卡片。顶部导航仍可继续引导，但画布里不会出现大块教程卡片。

如果你不想打开引导，也可以手动操作：

1. 点顶部导航里的 **商品/工作台**。
2. 点 **新建商品**。
3. 上传一张清楚的商品主图。
4. 填商品名，例如：`奶油白通勤托特包`。
5. 点 **创建并继续**。

你应该看到：页面进入这个商品的工作台，中间有几张可点击的卡片。

### 2. 补商品资料

1. 点击画布里的 **商品** 卡片。
2. 在右侧补充类目、价格、商品说明或本次想强调的方向。
3. 示例说明：`适合通勤和周末出门，轻便，大容量，奶油白配色。`
4. 点 **保存**。

你应该看到：表单保存成功。后面生成文案和图片时，会使用这些已保存资料。

### 3. 生成第一版文案

1. 点击 **文案** 卡片。
2. 在生成要求里写一句话，例如：

   ```text
   突出通勤、轻便、大容量，语气高级但不夸张。
   ```

3. 点 **运行当前节点**。如果你想从商品资料一路跑到图片，也可以点 **运行工作流**。

你应该看到：文案卡片生成标题、卖点、海报主标题和按钮文案。

如果不满意：只改一个方向再试，例如“更年轻一点”“更简洁一点”“少一点夸张词”。

### 4. 添加或连接参考图

如果你有想参考的风格图，可以这样做：

1. 选择或新增一个 **参考图** 卡片。
2. 上传一张参考图片，例如你喜欢的光线、背景、构图或风格。
3. 从参考图卡片边上的连接点拖到 **文案** 或 **生图** 卡片。

你只需要记住：**连过去 = 生成时参考它**。

你应该看到：画布上出现一条线。之后运行被连接的卡片时，会参考这张图的标签和图片信息。

### 5. 生成第一张图片

1. 点击 **生图** 卡片。
2. 确认 **生图** 卡片已经连到至少一个下游 **参考图** 卡片。生图卡片只负责触发生成，不自己展示/下载图片；生成结果会填进连过去的参考图卡片。
3. 写图片要求，例如：

   ```text
   白色托特包放在通勤桌面，旁边有笔记本电脑和咖啡，干净自然光，适合电商主图。
   ```

4. 点 **运行当前节点** 或 **运行工作流**。

你应该看到：下游参考图卡片被填入新图片，并在参考图卡片上提供预览和下载。

如果没有连接下游参考图卡片，系统会提示你先连接至少一个图片/参考图节点，不会把图片悄悄放在生图卡片上。

### 6. 继续调整，直到满意

每轮只改一两个点，更容易判断哪句话有效。

常见调整方式：

- 主体不清楚：加 `商品占画面中心，主体完整，纹理清晰`。
- 背景太乱：加 `干净背景，减少道具，只保留 1-2 个陪衬物`。
- 风格不对：加 `自然光`、`杂志感构图`、`极简电商`、`暖色生活方式`。
- 卖点没体现：把最重要卖点写进第一句话，例如 `大容量`、`轻便`、`可通勤`。

可复制的改写示例：

```text
背景更干净，只保留电脑和咖啡；包身纹理要清晰，阴影柔和。
```

满意后可以下载图片；如果还想连续微调，点顶部导航里的 **连续生图**。

### 7. 用连续生图做细节微调

1. 点顶部导航里的 **连续生图**。
2. 选择一个商品，或先自由生成。
3. 上传参考图或选中已有结果。
4. 用对话方式提出修改，例如：

   ```text
   保持包的角度不变，背景换成更明亮的办公室，减少桌面杂物。
   ```

5. 满意后，把图片回写到商品，后续工作台可以继续参考。

## 参考说明：工作台里有哪些卡片

这些说明是给已经跑通过一次流程、想更准确控制结果的用户看的。

### 商品

保存商品名、类目、价格和说明。下游生成会优先读取最新已保存的商品资料。

### 参考图

一个参考图卡片只放当前一张图。你可以手动上传，也可以让生图卡片把新图填进来。新图会替换这个卡片里的当前图；旧素材仍保留在商品历史里。

### 文案

生成标题、卖点、海报主标题和按钮文案。生成后可以在卡片里继续编辑，编辑后的文案会被后续生图使用。

### 生图

根据商品资料、文案、参考图和你写的图片要求触发图片生成。它不是图片槽位：生成出的图片会填入连接的下游参考图卡片；如果没有连接下游参考图卡片，运行会失败并提示先连接至少一个图片/参考图节点。

## 参考说明：连接和运行

- 把 A 连到 B：B 生成时会参考 A。
- 只想试一个卡片：选中卡片后运行当前节点。
- 想从商品资料一路生成到图片：运行整个工作流。
- 运行前先保存右侧表单；生成使用的是后端已保存内容。

## 参考说明：提示词配置

进入顶部导航 **配置**，找到 **提示词** 分组，可以调整四类长期默认提示词：

- `prompt_brief_system`：商品理解默认提示词。
- `prompt_copy_system`：文案生成默认提示词。
- `prompt_poster_image_template`：工作台生图模板。
- `prompt_image_chat_template`：连续生图模板。

建议用法：

- 单次想要什么效果：写在文案卡片或生图卡片里。
- 长期希望系统都保持某种口吻或格式：改配置页里的提示词。
- 不确定怎么改：先复制默认值，少量调整，再保存测试。

恢复默认会删除数据库里的自定义值，回到系统默认提示词。

常用占位符：

- 工作台生图模板：`product_name`、`category`、`price`、`source_note`、`instruction`、`title`、`selling_points`、`poster_headline`、`cta`、`size`、`kind`、`kind_label`、`kind_requirements`。
- 连续生图模板：`prompt`、`size`、`history_block`。

如果占位符写错，系统不会因为未知占位符直接崩掉，但那一段可能不会按预期替换。建议先小改、再测试。

## 参考说明：模型与运行时设置

顶部导航 **配置** 里还可以管理：

- 文案供应商和文案模型。
- 图片供应商和图片模型。
- 默认图片尺寸、允许图片尺寸。
- 上传文件大小限制。
- 文案/图片 provider 的 API key。

密钥字段不会回显。留空保存不会覆盖旧密钥；输入新值才会写入数据库。

## 常见问题

### 直接运行下游卡片，没有用上新资料？

先确认右侧表单已经保存。运行使用的是已保存内容，不是还没保存的输入框草稿。

### 生图卡片必须连接参考图卡片吗？

必须至少连接一个下游参考图卡片。生图卡片只负责触发和配置生成，图片预览/下载在被填充的参考图卡片上完成；一个生图卡片连接多个参考图卡片时，会并发生成并分别填充这些卡片。

### 图片效果不好，应该怎么改？

不要一次改很多句。每轮只改背景、构图、光线或主体细节中的一项。这样更容易知道哪句话让结果变好。

### 配置保存失败？

看页面错误提示里的字段名。常见原因是图片尺寸格式不对，例如应该写 `1024x1024`，或者默认尺寸不在允许尺寸列表里。

### 会不会在日志里记录完整提示词？

不应该。后端只保存必要的节点摘要和产物引用，不应记录完整提示词、密钥、上传字节或 provider payload。
