# ProductFlow Roadmap

这个路线图描述开源自托管版本的演进方向，不代表托管服务承诺。

## 当前阶段：开源自托管可运行

已完成的基础能力：

- FastAPI 后端、React/Vite 前端、PostgreSQL、Redis、Dramatiq worker。
- 单管理员登录和私有工作台。
- 商品创建、图片上传、参考图管理。
- 文案生成、编辑、确认和历史记录。
- 模板海报生成、AI 图片 provider 海报生成、海报下载。
- 连续图片会话和生成图挂回商品。
- 商品 DAG 工作流编辑、执行、持久化状态和恢复。
- 设置页管理 provider、模型、上传限制、任务重试等业务配置。
- 基础开源文件、MIT License、贡献/安全说明、issue/PR 模板。

## 近期优先级

### 1. 开发体验

- 补充更完整的本地部署截图和 troubleshooting。
- 增加一键 seed/demo 数据脚本。
- 增加可选 Dockerfile / compose app 服务示例。
- 让 release 脚本和自托管部署文档更通用，减少本机路径假设。

### 2. 测试与质量

- 扩展商品工作流 DAG 的端到端测试样例。
- 增加前端组件/交互回归测试策略。
- 为 provider mock、OpenAI Responses provider 和失败分类补更多边界测试。
- 为设置页 secret 更新和不回显行为补独立测试。

### 3. 工作流体验

- 改善 DAG 节点运行日志和失败原因展示。
- 增加节点级重试/跳过/复制能力。
- 优化大型商品详情页的状态刷新和局部 loading。
- 让图片会话和商品工作流之间的素材复用更直观。

## 中期方向

### 更丰富的输入

- 多来源商品信息导入。
- 商品 URL / 表格导入。
- 更结构化的品牌、受众、卖点输入。

### 更强的素材管理

- 素材收藏、标签、归档。
- 更多尺寸和平台适配。
- 可配置模板和品牌色/Logo。
- 更清晰的生成版本对比。

### Provider 扩展

- 更明确的 OpenAI 兼容 provider 配置示例。
- Provider 能力探测和健康检查。
- 按节点选择不同模型或 provider。
- 可插拔视频 provider 的接口探索。

## 长期探索

- 视频脚本、配音、字幕和模板渲染。
- 多成员协作和权限模型。
- 对象存储适配层。
- 部署包、容器镜像、Helm chart 或其他生产编排方案。
- 与外部店铺/投放平台的受控集成。

## 暂不计划

- 内置托管账号或代管模型密钥。
- 内置支付计费。
- 默认公开注册。
- 无人工确认的全自动投放链路。
