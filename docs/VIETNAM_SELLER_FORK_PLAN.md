## Problem Statement

Solo Vietnamese sellers launching products on Shopee and TikTok Shop waste roughly half a day per SKU preparing product content. The current workflow is messy:

1. Search other platforms for enough product images and product information.
2. Repaint or adapt the product visuals in Canva.
3. Write Vietnamese listing content manually.
4. Guess which image/title/description will improve click and buy conversion.
5. Manually upload everything into Shopee or TikTok Shop Seller Center.

The painful part is not only speed. The painful part is uncertainty: sellers do not know what content will make buyers click and buy.

## Demand Evidence

Current evidence is founder-pain plus observed peer pain:

- The founder personally spends about half a day preparing content for one new product.
- The founder reports many other Shopee/TikTok Shop sellers face a similar situation.
- The workflow already uses existing paid/effort-heavy substitutes: Canva, platform research, manual writing, and seller-center copy/paste.

This is enough to build a validation prototype. It is not enough yet to claim broad market demand. The first assignment is to test this on real sellers and real SKUs.

## Status Quo

Today the seller workflow is:

```text
Supplier / other ecommerce platform references
  -> collect product images and information manually
  -> adapt product visuals in Canva
  -> write Vietnamese title, description, selling points manually
  -> guess conversion angle
  -> copy/paste into Shopee and TikTok Shop Seller Center
```

Cost:

- Around half a day per SKU.
- Content quality depends on the seller's writing and visual taste.
- No clear system for learning which angles convert.
- Repeated work every time a new product is launched.

## Target User & Narrowest Wedge

### First target user

A solo Vietnamese seller who launches imported or sourced products on Shopee and TikTok Shop and needs to prepare product images, titles, descriptions, and promotional copy without knowing what will convert.

### First platform focus

Start with Shopee + TikTok Shop output modes, but do not integrate APIs in v1.

### Narrowest paid wedge

A **Vietnamese product launch kit generator**:

Input:

- Product name or rough category.
- Raw product images.
- Optional reference links or pasted competitor/supplier info.
- Target platform: Shopee, TikTok Shop, or both.
- Optional price, variant, promotion, shipping notes.

Output:

- Click-oriented Vietnamese title variants.
- Search keyword and hashtag suggestions.
- Short and long product descriptions.
- Selling point bullets.
- Buyer objection handling.
- Main-image concepts.
- Generated/repainted image candidates.
- Listing-image sequence plan.
- TikTok short-video/livestream hook.
- Manual upload checklist for Shopee/TikTok Shop.
- Export package: text copy, image files, and checklist.

## Premises

1. The first target user is a solo Vietnamese seller launching new SKUs on Shopee and TikTok Shop.
2. The first painful job is preparing launch content for one product, not managing the whole store.
3. The first version should not auto-publish to Shopee/TikTok Shop.
4. Keep ProductFlow's powerful canvas/workflow/image-session features intact, but make them Advanced Mode.
5. The product promise is conversion confidence, not just speed.
6. The first validation metric is whether real sellers reuse generated launch kits for real products.

## Approaches Considered

### Approach A: Guided Launch Kit on Top of ProductFlow

Recommended.

Summary: Keep ProductFlow's architecture and features. Add a default guided flow that produces Vietnamese platform-ready launch kits. Original canvas, image sessions, gallery, and settings remain available as Advanced Mode.

Effort: M
Risk: Low-Med

Pros:

- Fastest credible startup wedge.
- Reuses most of ProductFlow.
- Keeps advanced power for future sellers.
- Validates real seller content pain before platform API complexity.

Cons:

- Manual export at first.
- Requires strong content-quality rubrics.
- Needs UX simplification over a powerful existing product.

### Approach B: Marketplace Operating System

Summary: Build listings, inventory, orders, channel sync, fulfillment tasks, analytics, and marketplace connections.

Effort: XL
Risk: High

Pros:

- Bigger long-term company shape.
- Daily seller lock-in if it works.
- Opens paths to subscriptions and operations workflows.

Cons:

- Too broad for v1.
- Marketplace APIs, category schemas, auth, order state, and inventory sync dominate the work.
- Delays validation of content generation pain.

### Approach C: Canva Killer Visual Content Studio

Summary: Focus on image/poster generation and product visual repainting.

Effort: M-L
Risk: Med

Pros:

- Attacks the Canva pain directly.
- Very demoable.
- Produces visible before/after improvements.

Cons:

- Under-solves title/description/conversion uncertainty.
- Competes directly with Canva/CapCut/generic image tools.
- Leaves sellers to handle listing strategy elsewhere.

## Recommended Approach

Use **Approach A: Guided Launch Kit on Top of ProductFlow**.

The fork should preserve ProductFlow's power, but the default experience should feel like this:

```text
New Product
  -> Upload / paste references
  -> Choose Shopee / TikTok Shop / both
  -> Pick product category and buyer angle
  -> Generate launch kit
  -> Review title, description, image set, hooks, hashtags
  -> Export manually to Seller Center
  -> Track which kit was used and whether seller would reuse/pay
```

Advanced Mode remains:

```text
Open advanced canvas
  -> edit nodes
  -> run custom workflows
  -> use image sessions
  -> manage gallery
  -> tune providers/settings
```

## Product Shape

### New default navigation

### Core user flow

#### Step 1: Intake

Seller provides:

- Product name.
- Category.
- Target platform: Shopee, TikTok Shop, both.
- Raw product image.
- Optional reference images.
- Optional pasted supplier/competitor content.
- Optional price, variants, material, size, origin, shipping constraints.
- Desired brand tone: cheap deal, premium, practical, cute, trendy, trustworthy.

#### Step 2: Research synthesis

The system turns messy inputs into structured facts:

- What the product is.
- Who buys it.
- Main use cases.
- Buyer objections.
- Visual proof needed.
- Claims that are safe vs risky.
- Missing information seller must fill.

#### Step 3: Conversion-angle selection

Generate 3 angles:

1. Price/deal angle.
2. Problem/solution angle.
3. Lifestyle/trend angle.

Each angle includes:

- Why it might work.
- Buyer emotion targeted.
- Best platform fit.
- Risk or weakness.

Seller picks one primary angle or asks for all.

#### Step 4: Listing copy generation

Generate:

- Shopee title variants.
- TikTok Shop title variants.
- Short description.
- Long description.
- Bullet selling points.
- FAQ/objection handling.
- Search keywords.
- Hashtags.
- Promotion snippets.
- Livestream hook or short-video script.

#### Step 5: Visual kit generation

Generate or plan:

- Main thumbnail concept.
- Clean product image.
- Benefit image.
- Feature/detail image.
- Use-case image.
- Comparison/proof image if applicable.
- Promo image with price/discount copy.
- TikTok Shop/social cover image.

The system should output both generated images and editable prompts/layout notes.

#### Step 6: Export

Manual export package:

- `listing-copy.md`
- `shopee.txt`
- `tiktok-shop.txt`
- image files
- `upload-checklist.md`
- optional CSV-like structured export for future batch tooling

#### Step 7: Feedback loop

After seller uses a kit, capture:

- Was it used? yes/no
- Which title/image set was used?
- Did seller edit it?
- Would seller use it again?
- Would seller pay per kit/month?
- Optional platform metrics: impressions, clicks, add-to-cart, orders.

### Generation job

Use Dramatiq:

```text
LaunchKitGenerationTask
  -> parse source/reference content
  -> generate structured product facts
  -> generate conversion angles
  -> generate platform-specific copy variants
  -> generate image plan
  -> optionally enqueue image generation jobs
  -> persist variants and status
```

Reuse current async principles:

- DB state first.
- Queue is recoverable.
- Duplicate messages no-op.
- Status endpoint lightweight.
- Full detail refresh after completion.
- Keep task lifecycle status boring: `queued | running | succeeded | failed | cancelled`.
- Store generation detail in `progress_stage`, not in lifecycle status.
- Store retry/cancel flags and `failure_category` separately.

   - Support all seller categories through structured playbooks rather than a single first vertical.
   - Starter playbooks: fashion, cosmetics/beauty, electronics accessories, home goods, food, and other/custom.
   - Each playbook should define buyer objections, required visual proof, risky claims, suggested image sequence, content tone, and platform notes.

2. **Launch Quality Score**
   - Every Launch Kit gets a readiness panel.
   - Score dimensions: missing facts, title strength, image coverage, claim risk, buyer-objection coverage, platform fit, and generic wording risk.
   - Must be framed as publishing-readiness guidance, not guaranteed conversion lift.

3. **Store Profile / Seller Memory**
   - Add one single-admin Store Profile used by every launch kit.
   - Fields: store tone, target buyer, brand rules, color/logo notes, platform preferences, default shipping/promo notes, and prohibited claims.
   - This supports multi-category selling without losing store consistency.

4. **Durable LaunchKitGenerationTask**
   - Add a durable task table instead of relying only on `LaunchKit.status`.
   - Track status, attempt count, progress stage, failure category, retry/cancel flags, provider metadata, and timestamps.
   - Follow ProductFlow's existing async rule: database task first, Redis/Dramatiq dispatch second, duplicate worker messages no-op, startup recovery, and lightweight status polling.

5. **Named AI failure taxonomy and rescue behavior**
   - Define explicit errors for empty model responses, malformed JSON, missing generated sections, unsafe claims, provider refusal, rate limit, and timeout.
   - Each error must map to a task state, rescue action, seller-facing message, structured log context, and test case.

6. **Prompt-injection defense for pasted references**
   - Treat reference text as untrusted data, never instructions.
   - Add prompt boundaries, reference-as-data rules, output validation, risky-claim checks, and malicious-reference tests.

7. **Reference input budget policy**
   - Define max chars/counts for pasted text, URLs, images, and generation context.
   - Add visible warnings and reject/summarize behavior.
   - Do not silently truncate seller input.

8. **LaunchKit orchestration boundary**
   - LaunchKit coordinates ProductFlow systems; it must not become a parallel ProductFlow.
   - Product facts stay tied to `Product`; images stay in existing asset/storage flows; provider calls go through provider factories; generated images go through existing image/gallery flows; exports go through controlled storage.

9. **Lightweight LaunchKit status endpoint**
   - Add `GET /api/launch-kits/{id}/status`.
   - Return task status, progress stage, retry/cancel flags, failure category, queue position if available, counters, and `updated_at`.
   - Poll this endpoint while generation runs, then refresh full kit detail once terminal.

10. **Structured logs and basic metrics**
    - Add event logs for creation, queueing, generation start, stage completion, failure, completion, variant selection, export creation, quality-score computation, store-profile update, and playbook application.
    - Add counters/timers for kits created, generation duration, failures by category, exports by platform, quality score by category/platform, and current status.

11. **Seller-first Launch Kit detail hierarchy**
    - Top: publish readiness / Launch Quality Score.
    - Next: missing facts and required fixes.
    - Next: selected platform outputs: Shopee / TikTok Shop.
    - Next: image kit.
    - Next: export package.
    - Bottom: Advanced Mode + raw details.

### Visual Direction and Anti-Slop Rules

Design classifier: **APP UI**.

Direction: practical marketplace launch desk.

Rules:

- Product image and readiness score are the visual anchors.
- Use dense, calm layouts with strong typography and few colors.
- Use Shopee/TikTok platform cues only where they help platform distinction.
- Avoid purple/indigo AI gradients as the LaunchKit default identity.
- Avoid generic SaaS card grids, decorative icon circles, centered-everything layouts, emoji decoration, and vague AI-magic copy.
- Cards must be actionable: angle choice, variant choice, image candidate, export item.
- Copy must be utility language: orientation, status, next action.
- Never say or imply guaranteed conversion lift.

### Responsive and Accessibility Specs

| Viewport | Layout |
|----------|--------|
| Desktop >= 1024px | Dashboard uses paginated summary list. Create uses form + side context panel. Detail uses main review column + sticky export/status rail. |
| Tablet 768-1023px | Single column with sticky top readiness summary and in-page anchors. Export panel appears after platform outputs. |
| Mobile < 768px | Single column, sticky bottom primary action, segmented platform controls, quality score before outputs, Advanced Mode below seller actions. |

Accessibility requirements:

- Minimum 44px touch targets.
- Visible labels; never placeholder-only labels.
- Keyboard tab order follows visual order.
- Platform tabs use ARIA tab semantics and keyboard arrows.
- Generation progress updates use `aria-live="polite"`.
- Copy/export success and failure are announced to assistive tech.
- Focus returns to the triggering button after dialogs/download/copy actions.
- Color is never the only signal for score/risk/status.
- Body contrast target: 4.5:1 or better.
