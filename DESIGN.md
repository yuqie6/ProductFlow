# ProductFlow Vietnam LaunchKit Design System

Generated by `/plan-design-review` on 2026-06-21.

## Product Design Direction

**Practical marketplace launch desk.** The UI should feel like a focused seller workspace for preparing one SKU for Shopee and TikTok Shop, not a generic AI dashboard and not a Canva-style creative studio.

The seller should feel:

1. "I know what to do next."
2. "The system found what is missing."
3. "This content is ready enough to copy, edit, and upload."
4. "Advanced tools are still there when I need them."

## Core Principles

- Guided by default, powerful on demand.
- Readiness before outputs: show whether the kit is publishable before dumping generated copy.
- Product image is the visual anchor.
- Utility copy over hype copy.
- Manual export is a first-class workflow, not a workaround.
- Never imply guaranteed conversion lift.
- Show missing facts instead of hallucinating.
- Keep `/products` Advanced Mode reachable.

## Typography

Use a real Vietnamese-capable typeface stack, not the browser/system default as the primary design choice.

Recommended:

```css
--font-sans: "Be Vietnam Pro", "Inter", sans-serif;
--font-display: "Be Vietnam Pro", "Inter", sans-serif;
--font-mono: "JetBrains Mono", monospace;
```

If external fonts are not loaded in self-hosted mode, document the fallback clearly and keep spacing robust for Vietnamese diacritics.

## Color Tokens

Avoid purple/indigo AI-app gradients as the default identity. Existing ProductFlow indigo can remain in Advanced Mode, but LaunchKit should use practical ecommerce status colors.

```css
--color-bg: #f7f3ea;
--color-surface: #fffdf8;
--color-surface-muted: #f0eadf;
--color-text: #17202a;
--color-text-muted: #667085;
--color-border: #ded6c8;
--color-accent: #0f766e;
--color-accent-strong: #0b5f59;
--color-warning: #b45309;
--color-danger: #b42318;
--color-success: #15803d;
--color-shopee: #ee4d2d;
--color-tiktok: #111827;
```

Dark mode may reuse current ProductFlow dark surfaces, but must preserve 4.5:1 contrast for body text.

## Spacing, Radius, and Density

- Base spacing: 4px scale.
- Main content max width: 1200px for dashboard/create, 1360px for detail workspace.
- Cards/panels: `16-20px` radius. Do not use the same large bubbly radius everywhere.
- Touch targets: minimum 44px height/width.
- Dense but readable: LaunchKit is an app UI, not a marketing page.

## Component Rules

### Navigation

- Persistent `TopNav` remains.
- Primary nav order: Launch Kits, New Kit, Assets, Advanced, Settings, Help.
- Current route must be visually indicated with `aria-current="page"`.
- Advanced Mode is visible but secondary.

### LaunchKit dashboard

- First screen job: identify which kits need action.
- Product thumbnail + readiness status are the anchor.
- Use paginated summaries, not full kit cards.
- Empty state must include a warm explanation and primary action: create first kit.

### Create flow

- One primary action: create/generate kit.
- Visible labels, never placeholder-only labels.
- Reference budget warnings appear near the input that caused them.
- Category and platform choices must explain what changes in output.

### Detail page

Priority order:

1. Product header and readiness score.
2. Missing facts/fixes.
3. Platform output tabs.
4. Image kit.
5. Export panel.
6. Feedback.
7. Advanced Mode/raw details.

Cards are allowed only when the card is an action or selection unit: conversion angle, variant, export item, image candidate.

### Copy and export buttons

- Buttons must show copied/saved success state.
- Copy buttons need accessible labels naming the target, e.g. "Copy Shopee title".
- Export failure must show a retry path.

## Microcopy Tone

Vietnamese copy should be direct, seller-practical, and non-hype.

Good:

- "Còn thiếu chất liệu và kích thước. Bổ sung để mô tả đáng tin hơn."
- "Góc bán hàng này phù hợp TikTok Shop vì mở đầu bằng vấn đề khách gặp."
- "Nội dung đã sẵn sàng để copy, nhưng hãy kiểm tra lại claim về hiệu quả."

Avoid:

- "Tối ưu chuyển đổi chắc chắn"
- "AI siêu thông minh sẽ làm mọi thứ"
- "Unlock the power of ecommerce content"
- Generic ChatGPT-style explanations.

## Accessibility Rules

- Body text contrast at least 4.5:1.
- Visible focus rings for keyboard navigation.
- ARIA live region for generation status changes.
- Tabs must support arrow keys and proper roles.
- Upload zones must support keyboard activation and visible text alternatives.
- Copy/export actions restore focus and announce success/failure.
- Do not rely on color alone for quality score, risk, or status.

## Responsive Rules

### Desktop >= 1024px

- Detail page uses a two-column workspace: main review/output column plus sticky export/status rail.
- Dashboard uses table/list density, not a decorative card mosaic.
- Create page may use form + reference preview side panel.

### Tablet 768-1023px

- Detail page collapses to single column with sticky top status summary.
- Export panel follows platform outputs but remains reachable via anchor nav.

### Mobile < 768px

- Single-column layout.
- Sticky bottom action bar for the current primary action: continue, generate, copy/export, or retry.
- Quality score summary appears before generated content.
- Platform tabs become segmented controls with horizontal scroll only when necessary.
- Advanced Mode link moves below primary seller actions.

## Anti-Slop Rules

Do not use:

- Purple/violet/indigo gradient as the LaunchKit default identity.
- Generic 3-column feature grids.
- Decorative icons in colored circles.
- Centered everything.
- Emoji as UI decoration.
- Decorative blobs or wavy dividers.
- Generic hero copy like "unlock your ecommerce potential".
- Dashboard-card mosaics where a list/table would be clearer.
- "Conversion guaranteed" or similar claims.

## Existing ProductFlow Patterns to Reuse

- `TopNav` persistent navigation and breadcrumbs.
- `StatusPill` status language, adapted for LaunchKit readiness.
- `ImageDropZone` upload affordances.
- Product list pagination and loading skeleton patterns.
- Gallery image preview behavior for generated assets.
- TanStack Query status and error handling patterns.
